# Ad Hoc Workflow

Use this for unplanned requests.

## Classify First

Classify the request as tiny, durable context, spec change, active progress, feedback, conflict, emergency, or implementation.

## Process

1. Read the minimum local context needed.
2. Route durable facts into context or spec records.
3. Route implementation into progress only when requested or already active.
4. Route unclear feedback into review or validation.

## Stop

Stop when the request is handled or routed without creating hidden product truth.
