# Validation Workflow

Use this when assumptions, blockers, risks, QA feedback, or implementation claims need evidence.

## Evidence Rules

Accept official docs, source inspection, tests, reproducible command output, structured payloads, screenshots for UI behavior, and explicit user decisions.

Reject unsupported assumptions, UI-only proof of backend behavior, stale docs contradicted by current code, and "should be possible" claims.

## Process

1. Name the claim or question.
2. Choose the smallest reliable evidence source.
3. Record result, confidence, and impact on requirements, risks, decisions, or tasks.
4. Update spec records before using the finding for freeze or implementation.

## Stop

Stop when the claim is proved, disproved, blocked, or explicitly deferred.
