# MVP Assumptions

## ASM-001: Runtime Can Be Adapted

Status: proposed
Confidence: imported
MVP: yes
Phase: mvp
Source: `plans/product-brief.md`

OpenCode Runtime or Pi Runtime can provide stable enough session control, streaming events, tools, permissions, and configuration APIs behind an app-owned adapter.

## ASM-002: Provider Profiles Are Enough For MVP

Status: proposed
Confidence: imported
MVP: yes
Phase: mvp
Source: `plans/product-brief.md`

OpenAI-compatible provider profiles can satisfy the first provider/model management needs without provider-native APIs.

## ASM-003: Concurrent Runs Can Be Isolated

Status: proposed
Confidence: imported
MVP: unknown
Phase: mvp
Source: `plans/product-brief.md`

The app can support concurrent agent runs without mixing approvals, tool output, working directories, artifacts, or runtime events.
