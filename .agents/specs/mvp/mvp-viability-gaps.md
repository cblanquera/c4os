# MVP Viability Gaps

Status: ready-for-review
Updated: 2026-06-18

## Gaps

1. Runtime adapter viability needs proof before implementation freeze.
2. Browser scope needs a decision: isolated MVP preview, native browser after proof, or deferred post-MVP.
3. Terminal scope needs a decision: full interactive terminal, approved command/output surface, or deferred until backend lifecycle proof.
4. Concurrent sessions and runs need narrowing for the first MVP.
5. Extension inventory should remain post-MVP unless review proves it is required for the first usable workflow.
6. Wireframe behavior needs promotion into exact implementation acceptance before freeze.

## Recommended Path

Run a risk-first MVP review, then create POC records for runtime adapter, browser isolation, and terminal lifecycle before freezing implementation scope.
